## Algorithm Overview

### SFT (Supervised Fine-Tuning)

**Goal:** Fine-tune a pretrained model on (prompt, response) pairs to learn instruction-following.

**Data format (JSONL):** `{"prompt": "...", "response": "..."}`

**Core flow:**
1. `tokenize_prompt_and_output(prompt_strs, output_strs, tokenizer)`: concatenate and shift --
   `input_ids = concat[:-1]`, `labels = concat[1:]`. `response_mask` is 0 for prompt, 1 for response.
   SFT loss is only computed on mask=1 positions.
2. `get_response_log_probs(model, input_ids, labels)`: forward pass with log-sum-exp trick extracts
   per-token log P. Returns `{"log_probs": Tensor (B, seq_len)}`.
3. `sft_microbatch_train_step(policy_log_probs, response_mask, gradient_accumulation_steps)`:
   `loss = -masked_sum(log_probs, response_mask) / gradient_accumulation_steps`, backward.
4. Every `gradient_accumulation_steps` microbatches: `clip_grad_norm_(params, 1.0)`,
   `optimizer.step()`, `optimizer.zero_grad()`. Write loss to metrics.jsonl.

**Source mapping:**
- Training loop: `sft_train.py` main() -- remove wandb and vLLM eval sections
- Packed dataset (no padding): `sft_instruction_tune.py` PackedSFTDataset
- Helpers: `utils.py` (tokenize_prompt_and_output, get_response_log_probs, sft_microbatch_train_step, masked_normalize)

**Key adaptation from reference:** Reference calls `model(input_ids).logits` (HuggingFace CausalLMOutput).
`llm_foundry.common.model.Transformer` returns the logits tensor directly. Change ALL `.logits`
accesses throughout any ported code.

---

### DPO (Direct Preference Optimization)

**Goal:** Optimize policy with (prompt, chosen, rejected) preference data without explicit reward model.

**Data format (JSONL):** `{"prompt": "...", "chosen": "...", "rejected": "..."}`

**DPO Loss Formula:**
```
L = -log_sigmoid(beta * ( (log pi_theta(y_w|x) - log pi_ref(y_w|x)) - (log pi_theta(y_l|x) - log pi_ref(y_l|x)) ))
```
pi_ref: frozen reference model initialized same as pi_theta. beta: deviation penalty (typical 0.1).

**Core flow:**
1. Load pi_theta and freeze pi_ref (call `lm_ref.requires_grad_(False)` for all params).
2. For each (prompt, chosen, rejected): call `run_compute_per_instance_dpo_loss(lm, lm_ref, tokenizer, beta, prompt, chosen, rejected)`.
   - Formats texts with Alpaca template, encodes chosen/rejected
   - `get_sequence_log_prob(model, input_ids, device)` computes sum of log P over entire sequence
   - Returns scalar `-log_sigmoid(beta * logits_diff)`
3. Mean loss over batch, backward, optimizer.step(). Write metrics.

**Source mapping:** `dpo_train.py` contains `run_compute_per_instance_dpo_loss` with nested helper. Copy verbatim.

**Key adaptation:** `get_sequence_log_prob` calls `model(input_ids).logits` -- change to `model(input_ids)`.

---

### GRPO (Group Relative Policy Optimization)

**Goal:** Online RL alignment. Sample G responses per prompt, compute group-relative advantage to replace PPO value network.

**Data format (JSONL):** `{"problem": "...", "solution": "..."}` (MATH dataset style)

**GRPO flow (each step):**
1. **Rollout:** `self.inference.generate(prompts, n=group_size)` returns `list[list[str]]` shape [n_prompts, group_size].
2. **Flatten:** Expand to flat lists of length `n_prompts * group_size`.
3. **Reward + normalization:** `compute_group_normalized_rewards(reward_fn, flat_responses, flat_gts, group_size, advantage_eps=1e-6, normalize_by_std=True)`.
   Per group: `advantage_i = (r_i - mean) / (std + eps)`.
4. **(grpo_clip only) Cache pi_old:** forward pass under `torch.inference_mode()` to store old log_probs.
5. **Training:** gradient accumulation microbatches via `grpo_microbatch_train_step(policy_log_probs, response_mask, gradient_accumulation_steps, loss_type, advantages=...)`, clip grads (1.0), `optimizer.step()`.
6. **Hot-update inference:** `self.inference.update_weights(model.state_dict())` -- vLLM swaps weights, HF no-op.

**loss_type options (dispatched in compute_policy_gradient_loss in utils.py):**
- `no_baseline`: REINFORCE with raw rewards
- `reinforce_with_baseline`: REINFORCE + group-mean subtraction (recommended default)
- `grpo_clip`: PPO-clip with importance sampling (needs old_log_probs, cliprange=0.2)

**Source mapping:** Training loop: `grpo_train.py` main(). Core math: `utils.py` (all functions).

## Inference Backend Design

`llm_foundry/backends/inference.py` decouples GRPOTrainer from vLLM and HF generate.

**Auto-selection logic in `get_inference_backend(model_name, min_gpus=2)`:**
```
1. platform.system() != "Linux"          -> HFGenerateBackend (sentinel)
2. torch.cuda.device_count() < min_gpus  -> HFGenerateBackend (sentinel)
3. importlib.import_module("vllm") fails -> HFGenerateBackend (sentinel)
4. All pass                              -> VLLMBackend(model_name)
Silent fallback: never raises. backend_name records actual choice.
```

**Sentinel design:** Conditions 1-3 return `HFGenerateBackendSentinel(backend_name="hf_generate")`.
The sentinel stores `model_name` and raises `RuntimeError` if `generate()` is called before
`attach(model, tokenizer)`. GRPOTrainer calls `self.inference.attach(model, tokenizer)` right
after constructing its model instance.

**InferenceBackend interface (abstract base):**
```python
class InferenceBackend:
    backend_name: str  # "vllm" or "hf_generate"

    def generate(
        self,
        prompts: list[str],
        n: int = 1,
        max_new_tokens: int = 512,
        temperature: float = 1.0,
        stop: list[str] | None = None,
    ) -> list[list[str]]:
        # Returns result[i][j] = j-th response for prompts[i]
        raise NotImplementedError

    def update_weights(self, state_dict: dict) -> None:
        # vLLM: hot-swap weights in model_runner; HF: no-op
        raise NotImplementedError

    def attach(self, model, tokenizer) -> None:
        # HF sentinel: bind model+tokenizer; others: no-op
        raise NotImplementedError
```

**HFGenerateBackend implementation:**
- `__init__(self, model, tokenizer, device)`: store references
- `generate()`: loop n times per prompt, call `model.generate()` (via transformers), decode new tokens
- `update_weights()`: no-op (same model object used for training)
- `attach()`: no-op (already initialized with model)

> **[BUG-C6 修复] Tokenizer 接口统一说明**：`HFGenerateBackendSentinel.attach(model, tokenizer)` 接收的 tokenizer 必须实现简单接口：
> - `encode(text: str) -> list[int]`：将文本编码为 token ID 列表
> - `decode(ids: list[int]) -> str`：将 token ID 列表解码为文本
>

>     self,
>     input_ids: torch.Tensor,          # shape: (batch, seq)
>     max_new_tokens: int = 128,
>     do_sample: bool = True,
>     temperature: float = 1.0,
>     top_k: int = 50,
>     eos_token_id: int | None = None,
> ) -> torch.Tensor:
>     """自回归解码循环，兼容 HuggingFace generate() 最小接口。"""
>     device = input_ids.device
>     generated = input_ids.clone()
>     for _ in range(max_new_tokens):
>         logits = self(generated[:, -self._model_config.context_length:])
>         next_logits = logits[:, -1, :]  # (batch, vocab)
>         # [BUG-H5 修复] 处理 temperature=0 的贪婪解码情况，避免除零错误

>             next_token = next_logits.argmax(dim=-1, keepdim=True)
>         else:
>             if top_k > 0:
>                 values, _ = torch.topk(next_logits, top_k)
>                 next_logits[next_logits < values[:, -1:]] = float('-inf')
>             probs = torch.softmax(next_logits / temperature, dim=-1)
>             next_token = torch.multinomial(probs, num_samples=1)
>         generated = torch.cat([generated, next_token], dim=1)
>         if eos_token_id is not None and (next_token == eos_token_id).all():

- [ ] **Step 4: [BUG-4 修复] 修改 `backends/inference.py` — 实现真实模型调用**

**问题**：Plan 1 的 `HFGenerateBackendSentinel.generate()` 永远返回空字符串（`candidates = [""] * n`），导致 GRPO reward 全为零，训练完全无效。

**修复**：替换 `HFGenerateBackendSentinel.generate()` 实现，通过 `attach()` 存储的 tokenizer 正确分词，调用实际模型推理并解码：

```python
# backends/inference.py — HFGenerateBackendSentinel
def attach(self, model, tokenizer) -> None:
    """绑定模型和 tokenizer，激活推理能力。"""
    self._model = model
    self._tokenizer = tokenizer  # [BUG-4 修复] 同时存储 tokenizer，generate() 需要它

def generate(self, prompts, n=1, max_new_tokens=512, temperature=1.0, stop=None):
    if self._model is None or self._tokenizer is None:
        raise RuntimeError("attach() must be called before generate()")
    device = next(self._model.parameters()).device
    results = []
    for prompt in prompts:
        # 用 tokenizer 正确分词（而非字节编码），保证与模型 vocab_size 对齐
        enc = self._tokenizer(prompt, return_tensors='pt')
        input_ids = enc['input_ids'].to(device)
        candidates = []
        for _ in range(n):
            # temperature=0 时强制 greedy（避免 softmax 除以零）
            do_sample = temperature > 0
            output_ids = self._model.generate(
                input_ids,
                max_new_tokens=max_new_tokens,
                temperature=temperature if do_sample else 1.0,
                do_sample=do_sample,
            )
            new_ids = output_ids[0, input_ids.shape[1]:]
            text = self._tokenizer.decode(new_ids, skip_special_tokens=True)
            # 处理 stop sequences（保留 stop token，与 VLLMBackend 行为一致）
            if stop:
                for s in stop:
                    idx = text.find(s)
                    if idx != -1:
                        text = text[:idx + len(s)]
                        break
            candidates.append(text)
        results.append(candidates)
    return results
```

同时补全 `VLLMBackend.update_weights()`（见 §4.2 BUG-11 修复代码）。

2. 获取到 model/tokenizer 后调用 `self.inference.attach(model, tokenizer)`

- [ ] **Step 3: Implement `llm_foundry/stage5_align/sft.py`**

Port the `sft_train.py` main() loop to a class. Key design points:

1. `__init__(self, cfg)`: build `Transformer(cfg['model'])`, build `AdamW(...)`, set up output dir
2. **Byte Tokenizer 设计（修正）**：
   - `vocab_size = 256`（纯 byte tokenizer，每个字节一个 token，0-255）
   - `pad_token_id = 0`（空字节 `\x00`，不参与 loss 计算，attention mask 置 0）
   - `eos_token_id = 1`（保留字节 `\x01` 作为序列终止符）
   - 原设计 `vocab_size=10000` 存在问题：10000 词汇中仅 256 个实际使用，嵌入矩阵 97.4% 浪费
   - 最小 encoder/decoder 实现：
     ```python
     def byte_encode(text: str) -> list[int]:
         return list(text.encode("utf-8"))

     def byte_decode(tokens: list[int]) -> str:
         return bytes(t for t in tokens if t not in (0, 1)).decode("utf-8", errors="replace")
     ```
   - 使用方式：encode text as UTF-8 bytes（无需 HF tokenizer）；`configs/align.yaml` 中 `vocab_size: 256`
3. `self.model(input_ids)` returns logits directly (NOT `.logits` attribute)
4. SFT loss: `loss = -(log_probs * response_mask).sum() / max(mask_sum, 1.0)`, then backward
5. Gradient accumulation -> `clip_grad_norm_(params, 1.0)` -> `optimizer.step()`
6. Append `{step: int, loss: float}` to `{output_dir}/metrics.jsonl` each optimizer step
7. Save `model.save_checkpoint(str(output_dir / f'step_{step}.pt'))` every `checkpoint_every` steps plus final

Log-probs forward pass:
```python
logits = self.model(input_ids)  # (B, seq_len, vocab)
max_logits = logits.max(dim=-1, keepdim=True)[0]
log_probs_all = logits - max_logits - (logits - max_logits).exp().sum(-1, keepdim=True).log()
log_probs = log_probs_all.gather(-1, labels.unsqueeze(-1)).squeeze(-1)  # (B, seq_len)
```

- [ ] **Step 3: Implement `llm_foundry/stage5_align/dpo.py`**

Port `run_compute_per_instance_dpo_loss` from `dpo_train.py`. Key design:

1. Load policy model pi_theta and reference model pi_ref (same init weights, pi_ref frozen)
2. `get_sequence_log_prob(model, text_bytes)`: byte-level encoding, shift logits, compute sum of log P
3. DPO loss per sample: `-log_sigmoid(beta * (log_ratio_chosen - log_ratio_rejected))`
4. Mean over batch, backward, step

**Reference adaptation for `get_sequence_log_prob`:**
Original `dpo_train.py` calls `model(input_ids).logits`. Change to `model(input_ids)` (Transformer
returns logits tensor directly).

**Byte tokenization for DPO:** encode `prompt + response` as bytes, produce `input_ids` and `labels`
(shifted by 1), compute sum of log P over full sequence (both prompt and response tokens).
This matches the original design: DPO computes total log P(sequence), not just response tokens.

> ⚠️ **显存约束**：DPO 需同时持有 policy 模型（`pi_theta`）和 reference 模型（`pi_ref`），8G 显存仅支持极小模型（`d_model ≤ 256, n_layers ≤ 4`）。
>
> **推荐做法（cpu_ref 模式）**：将 `pi_ref` 常驻 CPU，计算 `log_prob` 时临时移至 GPU：
> ```python
> # DPOTrainer.__init__ 中：
> self.pi_ref = Transformer.from_checkpoint(ref_ckpt_path)
> if cfg.get("cpu_ref", True):  # 默认启用 cpu_ref 节省显存
>     self.pi_ref = self.pi_ref.cpu()
>
> # log_prob 计算时：
> pi_ref_gpu = self.pi_ref.to(device)
> with torch.no_grad():
>     ref_logprob = compute_logprob(pi_ref_gpu, batch)
> if cfg.get("cpu_ref", True):
>     pi_ref_gpu = pi_ref_gpu.cpu()
> ```
> 在 `configs/align.yaml` 的 DPO 配置中加入 `cpu_ref: true` 选项。

```python
# llm_foundry/stage5_align/grpo_utils.py

import re

def math_reward_fn(response: str, ground_truth: str) -> float:
    """Math reward function: 提取答案并对比"""
    # 从 response 中提取最后出现的数值（支持 LaTeX 格式如 $42$ 或 \(42\)）
    numbers = re.findall(r'(?:\$|\\\(|\\\[)?(-?\d+\.?\d*)(?:\$|\\\)|\\\])?', response)
    if not numbers:
        return 0.0
    try:
        pred = float(numbers[-1])
        gt = float(ground_truth)
        return 1.0 if abs(pred - gt) < 1e-3 else 0.0
    except ValueError:
        return 0.0

def dummy_reward_fn(response: str, ground_truth: str) -> float:
    """Dummy reward for testing"""
    return 0.0

# Reward function registry
REWARD_FUNCTIONS = {
    "math": math_reward_fn,
    "dummy": dummy_reward_fn,
}

def get_reward_fn(name: str):
    """Get reward function by name"""
    if name not in REWARD_FUNCTIONS:
        raise ValueError(f"Unknown reward_fn: {name}. Available: {list(REWARD_FUNCTIONS.keys())}")
    return REWARD_FUNCTIONS[name]
```

```python
def cmd_align(args) -> None:
    import yaml
    from llm_foundry.common.model import ModelConfig

    config_path = getattr(args, 'config', 'configs/align.yaml')
    with open(config_path) as f:
        raw_cfg = yaml.safe_load(f)

    # --method overrides align.method
    method = getattr(args, 'method', None) or raw_cfg.get('align', {}).get('method', 'sft')

    align_cfg = raw_cfg.get('align', {})
    method_cfg = align_cfg.get(method, {})
    model_raw = raw_cfg.get('model', {})

    model_config = ModelConfig(
        d_model=model_raw.get('d_model', 512),
        n_heads=model_raw.get('n_heads', 8),
        n_layers=model_raw.get('n_layers', 6),
        d_ff=model_raw.get('d_ff', 1344),
        context_length=model_raw.get('context_length', 256),
        vocab_size=model_raw.get('vocab_size', 10000),
        rope_theta=model_raw.get('rope_theta', 10000.0),
    )

    # [C-01 修复] 将 method 纳入 hash 计算，避免三种方法输出到同一目录互相覆盖
    import hashlib as _hashlib, json as _json
    cfg_with_method = dict(raw_cfg)
    cfg_with_method['__method__'] = method  # 将 method 纳入 hash
    cfg_str = _json.dumps(cfg_with_method, sort_keys=True, ensure_ascii=False)
    run_hash = _hashlib.sha256(cfg_str.encode()).hexdigest()[:8]
    output_dir = str(Path(raw_cfg.get('output', {}).get('base_dir', 'results')) / f'{run_hash}')

    cfg = {
        'model': model_config,
        'optimizer': {'lr': method_cfg.get('lr', 1e-5), 'weight_decay': 0.0},
        'training': {
            'max_steps': method_cfg.get('max_steps', 1000),
            'batch_size': method_cfg.get('batch_size', 8),
            'gradient_accumulation_steps': method_cfg.get('gradient_accumulation_steps', 1),
            'checkpoint_every': method_cfg.get('checkpoint_every', 500),
            'seed': method_cfg.get('seed', 42),
            'beta': method_cfg.get('beta', 0.1),           # DPO only
            'group_size': method_cfg.get('group_size', 8), # GRPO only
            'loss_type': method_cfg.get('loss_type', 'reinforce_with_baseline'),
        },
        'data': {'path': align_cfg.get('data_path', 'data/instructions.jsonl')},
        'output': {'dir': output_dir},
        'inference': raw_cfg.get('inference', {'min_gpus': 2}),
        'align': {'model_name': align_cfg.get('model_name', 'dummy')},
    }

    if method == 'sft':
        from llm_foundry.stage5_align import SFTTrainer
        trainer = SFTTrainer(cfg)
    elif method == 'dpo':
        from llm_foundry.stage5_align import DPOTrainer
        trainer = DPOTrainer(cfg)
    elif method == 'grpo':
        from llm_foundry.stage5_align import GRPOTrainer
        trainer = GRPOTrainer(cfg)
    else:
        print(f'[ERROR] Unknown align method: {method}. Choose sft | dpo | grpo')
        sys.exit(1)

    trainer.train()
```

## Testing Notes

### GRPO Backend Mock Test Design

The GRPO backend test (`test_stage5_grpo_backend` and the GRPO section of `test_stage5_align`)
uses `monkeypatch` or `unittest.mock.patch` to simulate three hardware environments:

**Path 1: Windows** (always HF generate)
```python
import sys
with patch.object(sys, 'platform', 'win32'):
    trainer = GRPOTrainer(cfg)
assert trainer.inference.backend_name == 'hf_generate'
```

**Path 2: Linux + insufficient GPUs**
```python
import sys
with patch.object(sys, 'platform', 'linux'):
    with patch.object(torch.cuda, 'device_count', return_value=1):
        trainer = GRPOTrainer(cfg)
assert trainer.inference.backend_name == 'hf_generate'
```

**Path 3: Linux + 2 GPUs + vllm available**
```python
import sys
with patch.object(sys, 'platform', 'linux'):
    with patch.object(torch.cuda, 'device_count', return_value=2):
        with patch('importlib.import_module', side_effect=mock_import_that_returns_vllm):
            with patch.object(inf_mod, 'VLLMBackend', return_value=mock_vllm):
                trainer = GRPOTrainer(cfg)
assert trainer.inference.backend_name == 'vllm'
```

Why mock `VLLMBackend` as well: even with vllm importable, the actual `VLLMBackend.__init__`
would try to start an LLM engine which requires real GPU hardware. Mocking the class prevents
this and allows the test to run on CPU/CI.

### SFT/DPO Numerical Accuracy

Tests do NOT verify loss values are correct (only that they are finite floats).
The correctness of the SFT/DPO math is validated by cross-referencing with
`reference_resource/Assignment5-alignment/` which contains the verified implementations.

### CPU-only Requirement

All test cases MUST run on CPU without GPU. Verify with:
```bash
CUDA_VISIBLE_DEVICES='' pytest tests/test_stages.py -v
```
